# Issue Tracker

Default issue tracker: local markdown.

Local issue directory:

```txt
docs/agents/issues/
```

Remote tracker:

```txt
Optional. This repo has a GitHub remote, but GitHub publishing is not part of the default workflow.
```

Remote publish rule:

```txt
Do not publish issues remotely unless the user explicitly asks for it.
```

Use local issue files for v0.1 pilots. Keep each issue small, vertical, and independently verifiable.
