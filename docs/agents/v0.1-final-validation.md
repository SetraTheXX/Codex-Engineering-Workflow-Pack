## Frontmatter Validation

All checked files follow this shape:

```yaml
---
name: <skill-name>
description: <trigger description>
---
```

Checks:

- Frontmatter includes only `name` and `description`.
- `name` equals folder name.
- No `disable-model-invocation`, `argument-hint`, provider metadata, or hook metadata.

Result: Pass.

## Description Overlap Check

Descriptions are distinguishable by workflow phase:

- `setup-codex-engineering-workflow`: repo-local configuration.
- `grill-with-docs`: plan clarification against docs/code/domain language.
- `to-prd`: convert clarified plan into PRD.
- `to-issues`: convert PRD/plan into vertical slice markdown issues.
- `diagnose`: reproduce and debug broken behavior.
- `tdd`: implement accepted behavior test-first.
- `zoom-out`: read-only code map.
- `prototype`: approved throwaway experiment.
- `improve-codebase-architecture`: architecture audit and refactor sequencing.
- `handoff`: session continuation summary.

Result: Pass.

## Reference Usefulness Check

References are short, one-level-deep, and useful as templates/checklists/patterns:

- Setup references define local docs layout and setup checklist.
- Diagnose references define diagnosis loop and feedback loop patterns.
- TDD references define red-green-refactor and test quality guidance.
- Grill references define question patterns, domain note shape, and ADR note format.
- PRD references define PRD template and quality checklist.
- Issues references define local issue template and vertical slice guide.
- Handoff references define template and quality checklist.
- Zoom-out references define code map checklist and module map template.
- Prototype references define logic/UI guides and exit checklist.
- Architecture references define audit checklist, deep module guide, report template, and sequencing guide.

No reference currently looks like unnecessary SKILL.md duplication.

Result: Pass.

## Windows Compatibility Check

Windows compatibility is preserved:

- Examples use PowerShell-friendly commands where commands appear.
- Paths are local markdown paths, not Unix-only absolute paths.
- Skills remind agents to quote Windows paths with spaces where relevant.
- No shell scripts were added.

Result: Pass.

## Local-first Check

Local-first behavior is preserved:

- Default issue output: `docs/agents/issues/`.
- Default architecture report output: `docs/agents/architecture/`.
- Default handoff output: configured local handoff path, fallback `docs/agents/handoff/`.
- PRD output is local markdown by default.
- GitHub or remote publishing is optional and requires explicit user request.

Result: Pass.

## Known Risks

- Trigger overlap should be tested in real Codex sessions, not only by static inspection.
- `setup-codex-engineering-workflow` is central; weak setup output could reduce usefulness of downstream skills.
- Some skills assume project docs may exist; pilot repos without docs will test fallback behavior.
- Architecture audit quality depends on real repo complexity and test surface visibility.
