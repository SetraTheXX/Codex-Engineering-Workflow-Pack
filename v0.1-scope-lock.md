## Neden Bu 10 Skill

Bu set uc ana akisi kapatir:

```txt
Fikir -> grill-with-docs -> to-prd -> to-issues -> tdd -> handoff
Bug -> diagnose -> regression test -> handoff
Repo -> zoom-out -> improve-codebase-architecture -> issue plan
```

Bu akislarda Codex'in en cok hata yaptigi noktalar kontrol altina alinir: belirsiz kapsam, feedback loop eksikligi, buyuk adimlar, domain dili kaybi ve oturum devri.

## `setup-codex-engineering-workflow` Neden Ilk Sirada

Butun diger skill'ler repo baglamina ihtiyac duyar. Setup skill'i once calismazsa her skill kendi varsayimini yapar ve paket dagilir.

Bu skill sunlari tanimlamalidir:

- Issue tracker tercihi: local markdown varsayilan, GitHub opsiyonel.
- Docs klasoru: PRD, plan, agent docs ve issue dosyalari nereye yazilacak.
- ADR klasoru: kararlar hangi klasorde tutulacak.
- `CONTEXT.md` veya esdeger domain language dokumani: domain terimleri nereden okunacak.
- Test komutlari: typecheck, lint, test, e2e gibi komutlar.
- Package manager: npm, pnpm, yarn, bun, uv, pip veya proje ozel komutlari.
- Agent handoff klasoru: oturum devri belgeleri nereye yazilacak.
- Local issue klasoru: varsayilan olarak `docs/agents/issues/`.

Gerekce: `diagnose` test komutlarini, `tdd` public interface ve test hedefini, `to-prd` docs hedefini, `to-issues` issue tracker'i, `handoff` devredilecek dosya yerini bu setup kararlarindan okumali.
