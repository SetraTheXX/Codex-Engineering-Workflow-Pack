# Setup Checklist

Use this checklist when preparing a repo for the Codex Engineering Workflow Pack.

## Explore

- Detect package manager and lockfiles.
- Read package scripts or equivalent task definitions.
- Find existing docs, plans, specs, ADRs, and domain language files.
- Check for local issue folders or remote issue conventions.
- Check for handoff or agent-note conventions.
- Identify files that already exist and must be preserved.

## Recommend

Record a proposed setup with these fields:

- Issue tracker
- Docs directory
- ADR directory
- Domain language document
- Package manager
- Test commands
- Local issue directory
- Handoff directory
- Files to create
- Files to update

## Ask

Ask only when evidence is ambiguous:

- Multiple package managers.
- No test command.
- Multiple docs roots.
- Existing remote issue tracker.
- Existing config file that needs merging.

## Write

After approval, create or update only agreed files. Preserve existing content.

## Verify

- Re-read created files.
- Confirm paths match the plan.
- Confirm no remote publish happened.
- Confirm unknowns are explicitly marked.
