# Domain Language Notes

Use this shape when proposing updates to domain language docs.

```md
## Term

Definition:

Use when:

Avoid:

Related terms:
```

## Rules

- Keep domain language free of implementation details.
- Prefer terms the product or business uses.
- Record ambiguity explicitly when it is not resolved.
- Do not add terms just because a helper function exists.
- Ask before writing to `CONTEXT.md` or another domain doc.
