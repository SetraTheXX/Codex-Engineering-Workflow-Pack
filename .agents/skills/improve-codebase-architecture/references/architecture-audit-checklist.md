# Architecture Audit Checklist

Use this checklist while auditing.

## Inspect

- Domain language docs.
- ADRs.
- Source modules.
- Callers and callees.
- Tests.
- Verification commands.
- Existing PRDs or issues.

## Evaluate

- Architectural friction.
- Deep module opportunities.
- Shallow module candidates.
- Interface complexity.
- Test surface gaps.
- Naming/domain mismatch.
- Risk and sequencing.

## Report

Each finding should include:

- files or modules,
- problem,
- proposed change,
- benefit,
- verification,
- risk level,
- recommended order.

## Avoid

- Big rewrite advice.
- Generic cleanup advice.
- Recommendations without file/module references.
- Recommendations without verification.
