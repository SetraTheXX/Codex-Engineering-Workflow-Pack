# Code Map Checklist

Use this checklist while mapping an unfamiliar area.

## Inspect

- User-named files or modules.
- Public entry points.
- Callers.
- Callees.
- Tests.
- Domain docs.
- ADRs.
- Verification commands.

## Report

- Focus area.
- Main flow.
- Important modules.
- Data and state movement.
- Side effects.
- Error paths.
- Test surface.
- Uncertainties.
- Recommended next skill.

## Avoid

- Editing code.
- Refactoring.
- Listing every helper.
- Guessing when references can be checked.
- Treating missing tests as passing tests.
