# Logic Prototype Guide

Use for state, rules, validation, and transformation questions.

## Shape

- Small local script or isolated module.
- Deterministic inputs.
- Printed state after each action.
- No persistence unless persistence is the experiment.

## Include

- Question.
- Input cases.
- Expected observations.
- Run command.
- Exit decision.

## Avoid

- New framework setup.
- Production integration.
- Hidden external services.
- Broad error handling.
- Polishing beyond the experiment.
