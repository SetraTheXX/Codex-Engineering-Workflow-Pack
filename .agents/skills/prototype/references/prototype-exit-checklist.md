# Prototype Exit Checklist

End every prototype with one decision.

## Discard

Use when the question is answered and the code should not survive.

Record:

- what was learned,
- files to remove,
- whether any note or ADR should remain.

## Promote To Implementation

Use when the idea is valid enough for production work.

Record:

- behavior to implement,
- acceptance criteria,
- risks,
- recommended next skill: `tdd` or `to-issues`.

## Revise

Use when the experiment was inconclusive.

Record:

- what blocked a decision,
- next experiment,
- new success criteria.
