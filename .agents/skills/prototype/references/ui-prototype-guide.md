# UI Prototype Guide

Use for layout, interaction, and visual option questions.

## Shape

- Temporary route, page, story, or local-only component variant.
- Clear label that it is a prototype.
- Visible state and variant controls.
- No production writes by default.

## Include

- Question.
- Variants to compare.
- How to open it.
- What to inspect.
- Exit decision.

## Avoid

- Hiding prototype code in production flow.
- Adding permanent navigation without approval.
- Depending on production data.
- Treating screenshots as the only validation.
