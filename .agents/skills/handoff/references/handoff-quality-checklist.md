# Handoff Quality Checklist

Use before writing or presenting a handoff.

## Must Include

- Current goal.
- Changed or created files.
- Decisions already made.
- Verification commands and results.
- Open questions.
- Risks.
- Next action.
- Useful paths.

## Must Avoid

- Secrets or credentials.
- Full copies of PRDs, issues, or large logs.
- Unverified success claims.
- Hidden assumptions.
- Vague "continue working" instructions.

## Good Handoff Test

A future session can answer:

- What is the goal?
- What has already been done?
- What should not be repeated?
- What needs verification?
- Where should work resume?
